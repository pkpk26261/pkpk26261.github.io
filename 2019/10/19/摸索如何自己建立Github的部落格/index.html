開始

Github repo 設定 & Hexo 搭建

至 https://github.com/ 申請一個帳號。
登陸之後，尋找一下建立倉庫的按鈕(頁面右上角的加號)，選擇 New repository (建立倉庫)

設置 Github 專案，建立部落格，會有兩種方式 :

  1. 以<個人帳號>.github.io為專案名稱，使用 master branch 建構，
    這樣生成的網址會是 https://<個人帳號>.github.io。

  2. 以自訂專案名稱 <倉庫名稱> ，使用 gh-pages banch 建構，
    這樣生成的網址會是 https://<個人帳號>.github.io/<倉庫名稱>。

我個人做的是想把網址簡單化，因此想保留 https://<個人帳號>.github.io 作為首頁網址，

步驟 1：新增一個 Github New repository

在個人的 Github 頁面上新增一個 New repository 。

步驟 2 : 安裝 Node.js 套件管理工具

  1. 安裝Git，至Git官網下載，安裝即可。

  2. 我們可以從Node.js 官網下載安裝，安裝完後至終端機輸入

    輸出版本後，便表示安裝已完成。
    個人我建議到這個步驟，重新開機一次！因為我曾經遇過一些狀況，若在 STEP 3 使用沒問題，可以不用重新開機。
     

步驟 3 : 安裝 Hexo

先打開cmd終端機，cd 到你想要安裝目錄，我個人是選擇在隨身硬碟的F:**槽裡安裝，方便之後換電腦維護編輯部落格。
若要換電腦維護編輯部落格，也是要將Hexo和Git、Hode.js安裝完成，也就是步驟 2和步驟 3的步驟要做完。
開始安裝Hexo**

hexo server 和 hexo s 是相同意思的，
輸入hexo s後，在網址列輸入http://localhost:4000，若網頁正常顯示，表示安裝成功。

接下來有幾個常用到的指令，這些指令將來進行網站修改會常常用到。

Hexo是使用MarkDown(MD)格式，每篇文章開頭基本上都一樣，
新文章加入以下內容：(---切記一定要加入)

文章類型：

  1. post：最常碰到的一般文章。
    新增的檔案會放在\source\_posts資料夾內建立[檔名].md的檔案。-
  2. page：不會出現在首頁中。
    基本上拿來撰寫about或tags之類的頁面。
    這類型會在\source\下建立檔名資料夾，並在檔名資料夾下新增index.md檔案。
  3. draft：草稿。
    如果文章太長可以先建立草稿，這樣就算發佈了訪客也看不到。
    草稿會存放在\source\_drafts底下，檔名為檔名.md。
    編輯完後，可以透過指令hexo publish draft 檔名讓文章公開。
    再來選擇一款合適自己的md編輯器，和使用一些常用md語法就能完成一篇文章了。
    個人我這邊都是使用 Visual Studio Code (VS Code) 編輯器。
